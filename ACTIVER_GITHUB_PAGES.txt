📚 Votre documentation MkDocs Material est maintenant prête !
Suivez ces étapes pour la publier en ligne comme CrushApp2025.

═══════════════════════════════════════════════════════════════════
  📌 ÉTAPE 1 : Activer GitHub Pages
═══════════════════════════════════════════════════════════════════

1. Allez sur votre repository GitHub :
   👉 https://github.com/thoumi/Automation

2. Cliquez sur "Settings" (en haut à droite)

3. Dans le menu de gauche, cliquez sur "Pages"

4. Dans "Build and deployment" :
   - Source : Sélectionnez "GitHub Actions"
   
5. Cliquez sur "Save"

═══════════════════════════════════════════════════════════════════
  📌 ÉTAPE 2 : Déclencher le Build
═══════════════════════════════════════════════════════════════════

La documentation va se construire automatiquement !

Option 1 : Attendre (automatique)
- Le workflow GitHub Actions se lance tout seul

Option 2 : Forcer le build (manuel)
```bash
git commit --allow-empty -m "Trigger docs build"
git push origin main
```

═══════════════════════════════════════════════════════════════════
  📌 ÉTAPE 3 : Vérifier le Build
═══════════════════════════════════════════════════════════════════

1. Allez sur votre repository

2. Cliquez sur l'onglet "Actions" (en haut)

3. Vous verrez le workflow "Deploy Documentation" en cours

4. Attendez 2-3 minutes (le temps du build)

5. Quand c'est ✅ vert, c'est prêt !

═══════════════════════════════════════════════════════════════════
  🎉 ÉTAPE 4 : Accéder à Votre Documentation
═══════════════════════════════════════════════════════════════════

Une fois le build terminé, votre documentation sera disponible à :

📖 https://thoumi.github.io/Automation/

Partagez ce lien avec votre équipe et vos clients ! 🚀

═══════════════════════════════════════════════════════════════════
  ⚙️ PERSONNALISATION (Optionnel)
═══════════════════════════════════════════════════════════════════

🎨 Modifier les Couleurs
Éditez `mkdocs.yml` :

theme:
  palette:
    primary: blue    # Changez la couleur principale
    accent: amber    # Changez la couleur d'accentuation

📝 Ajouter des Pages
1. Créez un fichier dans `docs/` :
   exemple : docs/nouvellepage.md

2. Ajoutez-le dans `mkdocs.yml` :
   nav:
     - Ma Page: nouvellepage.md

3. Commit et push :
   git add docs/nouvellepage.md mkdocs.yml
   git commit -m "Ajout nouvelle page"
   git push origin main

Problème : Le workflow échoue
Solution : 
1. Allez dans Actions → Deploy Documentation
2. Cliquez sur le workflow en échec
3. Consultez les logs pour voir l'erreur
4. Souvent : erreur de syntaxe dans mkdocs.yml

Problème : Page blanche
Solution :
1. Vérifiez que le dossier s'appelle bien "Automation" (pas "automation")
2. Attendez 5-10 minutes (propagation DNS)
3. Effacez le cache de votre navigateur (Ctrl+Shift+Del)

Problème : 404 Not Found
Solution :
1. Vérifiez Settings → Pages : Branch doit être "gh-pages"
2. Le dossier doit être "/ (root)"
3. Repoussez un commit pour redéclencher le build
